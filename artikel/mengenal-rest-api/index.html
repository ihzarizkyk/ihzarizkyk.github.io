Mengenal Rest Api

REST API

REST API atau singkatan dari representational state transfer application programming interface adalah arsitektur yang berada didalam api. dengan menggunakan REST, anda bisa mengakses data dari REST Server menggunakan Metode HTTP POST, PUT, DELETE, OPTIONS untuk bisa mengambil data / resource tertentu dari REST Server yang berbeda.

REST dan HTTP tidaklah Sama

Padahal, karena REST juga bermaksud untuk membuat web (internet) lebih efisien dan standar, dia menganjurkan untuk menggunakan prinsip REST dengan lebih ketat. Dan dari situlah orang mencoba untuk mulai membandingkan REST dengan web (HTTP). Roy fielding, dalam disertasinya, tidak menyebutkan petunjuk implementasi apapun - termasuk preferensi protokol dan HTTP. Hingga saat ini, Anda menghormati 6 prinsip panduan REST, Anda dapat memanggil antarmuka Anda RESTful.

Sederhananya, dalam gaya arsitektur REST, data dan fungsionalitas dianggap sebagai sumber daya dan diakses menggunakan Uniform Resource Identifier (URI). Sumber daya ditindaklanjuti dengan menggunakan serangkaian operasi sederhana dan terdefinisi dengan baik. Klien dan server bertukar representasi sumber daya dengan menggunakan antarmuka dan protokol standar - biasanya HTTP.

Contoh REST API Sederhana Menggunakan PHP :

<?php

$objek = [
'Nama'=> 'Mochammad Ihza Rizky Karim',
'Tautan'=> 'ihzarizkyk.space',
'data'=> [
'Kunci1'=> 'Nilai1',
'Kunci2'=> 'Nilai2',
'Kunci3'=> 'Nilai3'
]
];
header("content-type: application/json");
echo json_encode($objek);

?>

Hasilnya :

{"Nama":"Mochammad Ihza Rizky Karim","Tautan":"ihzarizkyk.space","data":{"Kunci1":"Nilai1","Kunci2":"Nilai2","Kunci3":"Nilai3"}}
