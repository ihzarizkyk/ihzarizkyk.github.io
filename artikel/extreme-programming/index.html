Agile Extreme Programming (XP)

Extreme Progamming atau Biasa disingkat XP adalah Framework turunan dari Metodologi Agile. Metode ini dikenalkan oleh Kent Back sebagai metode pengembangan software development yang cepat, efisien, beresiko rendah, fleksibel, terprediksi, saintifik, dan menyenangkan. XP cenderung menggunakan pendekatan Object Oriented yang berisi proses - proses sebagai berikut :

  • Pertama, Plan yaitu proses perencanaan sebuah pengembangan aplikasi atau produk mulai memikirkan kebutuhan-kebutuhan untuk pengembangan (Seperti : User Story)
  • Kedua, Design yaitu membuat sebuah gambaran sample atau prototype sebagai acuan dalam pengembangan selanjutnya / Merancang class object karena berhhubungan dengan Object oriented Programming (Menggunakan Class Diagram, Use Case)
  • Ketiga, Coding proses core yang mulai membangun sistem sebuah aplikasi dan mengembangkannya dengan menerapkan design sebelumnya
  • Keempat, Testing yaitu ketika proses pengembangan sudah hampir dirilis, maka perlu pengujian sehingga aplikasi yang dikembangkan memang betul layak dirilis ke publik

Extreme Programming hanya menyasar untuk sebuah tim yang kecil dengan anggota sedikit untuk menghadapi Requirements yang belum jelas karena perubahan - perubahan yang ada.

Core Value dari Extreme Programming :

  • Communication, Komunikasi menjadi hal yang penting agar menjalin hubungan baik dengan Tim, Manajer Proyek, Developer dan Pelanggan
  • Feedback, Selalu mengevaluasi setiap pengembangan produk yang sedang berjalan. Secara rutin membahas kesalahan dalam pengembangan produk atau aplikasi agar mendapatkan solusi yang tepat. Indikator dari perkembangan produk atau proyek adalah feedback
  • Simplicity, Membuat semua proses yang berjalan menjadi lebih mudah serta sesederhana mungkin. Mengembangkan sebuah fitur yang dirasa perlu untuk kedepannya.
  • Courage, Mendorong keberanian para Developer dalam sebuah tim untuk bereksperimen dalam proses pengembangan Sebuah produk atau aplikasi dan membangun komunikasi dengan Anggota Lain.

Lalu, untuk kelebihan dan kekurangannya menurut saya sebagai berikut :

Kelebihan :

  • Meningkatkan Kepuasan Pelangggan / User
  • Mempercepat pembangunan sistem
  • Menjalin Komunikasi yang baik dengan User / Pelanggan
  • Meningkatkan Empati antar Developer atau Programmer

Kekurangan :

  • Cerita atau Requirements yang tidak jelas atau detail diawal
  • Tidak bisa membuat Kode yang detail diawal
  • Tidak memiliki Aturan Baku dalam Dokumentasi (Tidak Formal)

Semoga Bermanfaat
