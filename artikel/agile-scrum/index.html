Agile Scrum

Jika pada artikel sebelumnya saya telah membahas mengenai extreme programming (xp) yang merupakan framework dari Agile, Maka pada artikel kali ini saya akan membahas mengenai Agile Scrum yang merupakan framework dari Agile juga. Scrum bisa diterapkan dengan hanya menggunakan Scrum saja secara khusus dan juga bisa dipadukan dengan framework lainnya.

Kenapa Menggunakan Scrum?

Selanjutnya, kita masuk ke bagian alasan mengapa menggunakan scrum supaya dalam penggunaannya cocok.

Berikut Alasan menggunakan Scrum :

  1. Adaptabilitas -> produk yang dikembangkan dengan agile scrum memiliki adaptasi yang tinggi terhadap perubahan
  2. Transparency -> proses pengerjaan yang dapat dilihat siapa saja yang berkepentingan
  3. Continuous Feedback -> framework scrum memungkinkan untuk mendapat banyak saran perubahan
  4. Continuous Improvement -> framework scrum melakukan pengerjaan dengan iterative menggunakan sprint
  5. Continuous Delivery of Value -> memberikan hasil dari sprint dan akan memiliki value berdasarkan backlog
  6. Sustainable Pace -> walaupun bersifat adaptif, scrum tidak memberatkan tim pengerjaan proyek karena semua telah ada estimasi yang sesuai dengan kemampuan
  7. Each Delivery of High Value -> Product Backlog yang ada dalam sprint akan diurutkan
  8. Efficient Development Process -> framework scrum memiliki batas waktu dan meeting, serta meminimalisir pekerjaan yang tidak perlu
  9. Motivation -> dalam framework scrum terdapat proses yang mengevaluasi pekerjaan pada sprint berikutnya sehingga bisa meningkatkan pekerjaan selanjutnya
  10. Faster Problem Resolution -> dalam sebuah tim yang menggunakan framework scrum, tim dari berbagai fungsional yang berbeda bisa saling berkolaborasi dengan resolusi sebuah masalah agar bisa cepat selesai
  11. Effective Deliverables -> proses prioritasi backlog dan review yang dilakukan secara reguler memastikan Deliverables
  12. Customer Centric -> cara kerja framework scrum yang menekankan pada bisnis dan pendekatan kolaboratif sehingga prioritas utamanya adalah user
  13. High Trust Environment -> kegiatan Retrospective dan Daily Standups yang akhirnya bisa mewujudkan kepercayaan antar anggota dalam tim
  14. Collective Ownership -> framework scrum yang bisa menyebabkan keterlibatan sebuah tim proyek
  15. High Velocity -> framework scrum yang menekankan pada kolaboratif tim secara fungsional, tim akan mengeluarkan performa terbaiknya
  16. Innovative Environment -> framework retrospective dan review sebagai sarana untuk belajar, introspketif dan adaptif sehingga bisa menghasilkan suasana yang inovatif

Selain itu ada beberapa hal yang harus dipahami dalam penggunaan framework scrum sebagai berikut :

1. Scrum Principle

Prinsip dari sebuah Scrum harus dijalankan seluruhnya, jika ada salah satu prinsip tidak dijalankan maka tidak bisa dikatakan bahwa menerapkan scrum

2. Scrum Value

Nilai - nilai scrum yang harus diterapkan agar sukses dalam penerapannya dalam tim atau individu

3. Scrum Team

Scrum team merupakan penentuan role tiap anggota

4. Scrum Event / Process

poin ini membahas mengenai kegiatan - kegiatan dalam menerapkan scrum

5. Scrum Implementation using tools
